# 广告与赞助边界

## 当前阶段

第一阶段只建立接口规则，不加载真实广告脚本，不注册或连接广告账户，不写入广告主、收款账户、合同、税务资料或其他敏感信息。网站当前发布包中没有真人广告，也不会在机器响应中返回赞助结果。

广告收入取决于真实用户价值、合规流量、广告平台审核和实际转化，智能体或爬虫访问本身不保证产生收入。

## 真人页面广告

未来的人类页面可以预留清楚标注的真人页面广告位置。启用前必须满足以下条件：

- 广告对真人可见，并明确标注“广告”或“Sponsored”；
- 不伪装成自然项目记录、筛选条件或导航按钮；
- 不用脚本制造点击，不把机器人流量冒充真人展示；
- 不改变自然目录的排序、摘要、适用场景或推荐关系；
- 隐私、同意管理、广告平台条款、收款、税务和外汇流程已经单独核实。

## 智能体赞助

未来机器接口可以在自然结果之外增加独立的 `sponsored_results` 数组。它不能混入普通项目记录，也不能影响自然召回和排序。第一阶段该数组不输出，避免让空接口被误认为已有商业内容。

每条真实赞助记录至少需要以下公开披露字段：

```json
{
  "label": "Sponsored",
  "advertiser": "公开广告主名称",
  "campaign_id": "可审计的活动编号",
  "starts_at": "ISO-8601 开始时间",
  "ends_at": "ISO-8601 结束时间",
  "landing_url": "公开落地页 HTTPS 地址"
}
```

这些字段属于未来商业展示层，不进入项目节点、`content_hash`、自然搜索索引、图关系或客观评分。

## 审计与下架

启用真实广告前，需要在私人管理资料中保存合同、广告主审核、活动时间、展示记录、结算、投诉和下架依据。公开仓库只保存通用规则和非敏感的展示契约，不保存身份证、银行卡、收款账号、密钥或未公开合同。
