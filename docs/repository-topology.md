# 总仓库与分类仓库

## 当前结构

当前只有一个公开总仓库：

```text
Zunzhe966/kai-yuan-da-shu-li
```

它同时保存分类规范、项目节点、关系、网站构建器、机器索引和贡献规则。当前提交基线规模很小，不为未来容量提前创建空仓库。

## 未来拆分

满足以下任一条件时，才考虑把一个领域拆成独立公开仓库：

- 总仓库接近 GitHub 建议的 10GB 仓库大小；
- 完整验证或 Cloudflare 构建时间明显影响更新；
- 一个领域形成稳定的独立维护者和贡献流程；
- 领域数据需要单独版本发布和回滚。

分类仓库使用同一套字段规范，并发布固定提交。总仓库的 `data/sources.yaml` 记录仓库地址、固定提交和数据路径。构建器只接受公开 HTTPS 来源，不接受私人仓库，也不使用 Git 子模块。

## 私人仓库

私人项目清单不属于开源目录数据。它保存在私人开发同步仓库或本机管理目录，不写入 `data/sources.yaml`，不进入 Cloudflare 构建，也不向外部智能体披露。

## Cloudflare关系

Cloudflare Pages 只连接公开总仓库。未来分类仓库由总仓库的验证流程读取固定版本并合并成发布包；用户不需要在 Cloudflare 中逐个授权几十个仓库。
